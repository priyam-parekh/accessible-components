context

In modern UI design, accessibility plays a huge role in ensuring that all users can interact with the interface

This case study analyzes how search boxes function across 3 different applications: Things, Notion, Gmail

Notion

I chose these 3 programs because they are well-renowned for having great design and inclusivity

I wanted to evaluate the validity of these claims through the lens of accessibility

reflection

• In terms of usability and accessibility, all 3 platforms (Things, Notion, and Gmail) faired well as they were easy to navigate, and use, and displayed the functionality to the user. For instance, Things was incredibly intuitive when it came adding and viewing the different task lists; however, it was not the best for accessibility since some parts of the application were difficult to navigate to using focus order and screen reader. In my design, I used Thing's user interface and Gmail's accessibility into account to meet both aspects of design.

• The accessibility considerations that were not not addressed in the components was discoverability. It is important that the user can discover the functionality of the interface—resulting in me including automatic suggestions and an info section in my design.

• Based on the video, "mismatch" refers to the gap between a person's abilities and the design of a particular product; I effectively bridge this gap by allowing non-tech savvy individuals use the search box with automatic suggestions and improving discoverability.

• An example of how the inaccessibilitity of Notion's search box (input) when using a mouse/trackpad could negatively impact users with impairments is that it is placed in the side bar, essentially concealed—this could make it harder for a screen reader to detect it. Another example which relates to output is with the inaccessibility of Gmail's drag and drop function which doesn't show the user where a email could be dropped to.

• Mouse and keyboard would most commonly be prioritized when creating user interface components because most websites are accessed on a laptop/desktop. This impacts the user experience by having icons and text more smaller on the screen because a mouse is more precise; this results in a site on a mobile device being smaller and harder to press and navigate.
